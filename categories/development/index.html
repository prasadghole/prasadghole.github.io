Five Principals for of embedded security Message Off course this is what you want to protect from intruder. Integrity Original message sent from sender should not altered when received by receiver. Authentication Message should be received from intended sender party. Authorization Sender of message should have be authorized to execute some action on receptor Anti-replay A valid message should not be used again by intruder to carry out malicious activities
