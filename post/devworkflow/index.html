If you're working on embedded systems development for STM32 microcontrollers, you'll need an efficient build automation tool to streamline your software development process. In this blog post, we'll explore how to create a CMake build file using the GCC toolchain, specifically designed for STM32 microcontrollers.

## Prerequisites

Before we begin, make sure you have the following tools set up:

  • CMake: A cross-platform build automation tool.
  • GCC Toolchain: The GNU Compiler Collection toolchain specifically for STM32 microcontrollers.
  • STM32Cube: The software development framework for STM32 microcontrollers.

## Project Setup

Assuming you have a basic project structure in place, with source code files located in the `src` directory and header files in the `inc` directory, follow these steps to create the CMake build file:

### Step 1: Create CMakeLists.txt

In the project root directory, create a file named `CMakeLists.txt`. This file will contain the instructions for building your project using CMake.

### Step 2: Set the Toolchain File

Inside the `CMakeLists.txt` file, specify the path to the STM32 toolchain file using the `CMAKE_TOOLCHAIN_FILE` variable:

```cmake set(CMAKE_TOOLCHAIN_FILE "path/to/STM32Toolchain.cmake") ```

Replace `"path/to/STM32Toolchain.cmake"` with the actual path to the STM32 toolchain file for GCC.

### Step 3: Define Microcontroller Parameters

Set the target microcontroller family and the specific microcontroller model using the `MCU_FAMILY` and `MCU` variables:

```cmake set(MCU_FAMILY "STM32F4xx" CACHE STRING "Target STM32 microcontroller family") set(MCU "STM32F407VG" CACHE STRING "Target STM32 microcontroller") ```

Adjust these variables to match your target microcontroller's specifications.

### Step 4: Add Source Files

List your source files in the `SOURCES` variable:

```cmake set(SOURCES src/main.c src/gpio.c

) ```

Make sure to include all the necessary source files for your project.

### Step 5: Specify Include Directories

Specify the directories containing the header files using the `include_directories` command:

```cmake include_directories( inc

) ```

Include any additional directories where your header files are located.

### Step 6: Create the Executable

Create the executable by adding the following command:

```cmake add_executable(${PROJECT_NAME} ${SOURCES}) ```

### Step 7: Set Compiler and Linker Flags

Set the compiler flags using the `target_compile_options` command:

```cmake target_compile_options(${PROJECT_NAME} PRIVATE ${MCU_FLAGS}) ```

Specify the linker flags and dependencies using the `target_link_options` command:

```cmake target_link_options(${PROJECT_NAME} PRIVATE -T${CMAKE_CURRENT_SOURCE_DIR}/STM32F4xx.ld ${MCU_FLAGS} ) ```

Make sure to adjust the linker flags and dependencies to match your target microcontroller's requirements.

### Step 8: Specify Output Binary Format

Specify the output binary format (ELF, HEX, BIN) and

add a custom command for generating the HEX and BIN files after the build:

```cmake set(HEX_FILE ${PROJECT_NAME}.hex) set(BIN_FILE ${PROJECT_NAME}.bin)

add_custom_command(TARGET ${PROJECT_NAME} POST_BUILD COMMAND ${CMAKE_OBJCOPY} -Oihex $<TARGET_FILE:${PROJECT_NAME}> ${HEX_FILE} COMMAND ${CMAKE_OBJCOPY} -Obinary $<TARGET_FILE:${PROJECT_NAME}> ${BIN_FILE} COMMENT "Converting ${PROJECT_NAME} to ${HEX_FILE} and ${BIN_FILE}" ) ```

### Step 9: Build the Project

You're now ready to build your project using CMake. Open your terminal, navigate to the project directory, and run the following commands:

```bash mkdir build && cd build cmake .. make ```

The build process will generate the necessary object files and the final executable.

Congratulations! You've successfully created a CMake build file for your STM32 microcontroller project using the GCC toolchain. With CMake, you can easily manage dependencies, compile your code, and generate the required output binary files.
