Daily Notes - May 26, 2023

Today, I worked on creating a CMake build file for STM32 microcontrollers using the GCC toolchain. It was an exciting experience, and I want to share my process with you.

![STM32 Microcontroller](/images/post/stm32_microcontroller.png)

Prerequisites

Before starting, ensure that the following tools are set up:

  • CMake: A cross-platform build automation tool.
  • GCC Toolchain: The GNU Compiler Collection toolchain specifically for STM32 microcontrollers.
  • STM32Cube: The software development framework for STM32 microcontrollers.

[[/static/images/post/home-made-robot-desk.jpg]

Project Setup

I began by setting up my project structure with the necessary directories. The source code files are located in the `src` directory, and the header files are in the `inc` directory.

![Project Structure](path/to/project_structure.png)

Step 1: Create CMakeLists.txt

To create the CMake build file, I created a new file named `CMakeLists.txt` in the project root directory. This file will contain the instructions for building the project using CMake.

Step 2: Set the Toolchain File

Inside the `CMakeLists.txt` file, I specified the path to the STM32 toolchain file using the `CMAKE_TOOLCHAIN_FILE` variable:

```cmake set(CMAKE_TOOLCHAIN_FILE "path/to/STM32Toolchain.cmake") ```

![Setting Toolchain File](path/to/toolchain_file.png)

Step 3: Define Microcontroller Parameters

Next, I defined the target microcontroller family and the specific microcontroller model using the `MCU_FAMILY` and `MCU` variables:

```cmake set(MCU_FAMILY "STM32F4xx" CACHE STRING "Target STM32 microcontroller family") set(MCU "STM32F407VG" CACHE STRING "Target STM32 microcontroller") ```

![Defining Microcontroller Parameters](path/to/mcu_parameters.png)

Step 4: Add Source Files

I listed the source files in the `SOURCES` variable within the `CMakeLists.txt` file:

```cmake set(SOURCES src/main.c src/gpio.c

) ```

![Adding Source Files](path/to/source_files.png)

Step 5: Specify Include Directories

To specify the directories containing the header files, I used the `include_directories` command in the `CMakeLists.txt` file:

```cmake include_directories( inc

) ```

![Specifying Include Directories](path/to/include_directories.png)

Step 6: Create the Executable

To create the executable, I added the following command to the `CMakeLists.txt` file:

```cmake add_executable(${PROJECT_NAME} ${SOURCES}) ```

Step 7: Set Compiler and Linker Flags

I set the compiler flags using the `target_compile_options` command and specified the linker flags and dependencies using the `target_link_options` command within the `CMakeLists.txt` file.

Step 8: Specify Output Binary Format

To specify the output binary format and generate the HEX and BIN files after the build, I added a custom command to the `CMakeLists.txt` file:

```cmake set(HEX_FILE ${PROJECT_NAME}.hex) set(BIN_FILE ${PROJECT_NAME}.bin)

add_custom_command(TARGET ${PROJECT_NAME} POST_BUILD COMMAND ${CMAKE_OBJCOPY} -Oihex $<TARGET_FILE:${

PROJECT_NAME}> ${HEX_FILE} COMMAND ${CMAKE_OBJCOPY} -Obinary $<TARGET_FILE:${PROJECT_NAME}> ${BIN_FILE} COMMENT "Converting ${PROJECT_NAME} to ${HEX_FILE} and ${BIN_FILE}" ) ```

Step 9: Build the Project

I built the project by navigating to the project directory in my terminal and running the following commands:

```bash mkdir build && cd build cmake .. make ```

The build process generated the necessary object files and the final executable.

![Build Output](path/to/build_output.png)
