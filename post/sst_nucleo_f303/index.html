Porting SST on Nucleo F303

The Quantum Framework, developed by Miro Samek, is a software framework designed for building event-driven, reactive embedded systems. It provides a structured approach to software development, emphasizing modularity, reusability, and scalability.

At the core of the Quantum Framework is the concept of hierarchical state machines (HSMs), which are used to model the behavior of the system. HSMs allow for the representation of complex system behaviors in a hierarchical and modular manner, making it easier to understand and maintain the software.

Super Simple Tasker

Instead of moving to this full fledge framework from bare metal programs. Super simple tasker also known as hardware based RTOS. Using nested vectore controller features to simulate preemptive kernel, also saving overhead of private stack for each task.

SST Template application

Design

Active objects Identification

We have to first identify tasks or active objects in our project. Lets say for simplicity we have LED driver turning LED on or off. So we can wrap LED driver in task, which takes input events LED_ON and LED_OFF.

Signals and events

Enumerate all the signals this AO is going to handle. We can have globally identified events so that all the AO can communicate with each other with central event repository.

  enum Signals {
    LED_ON,
    LED_OFF,
    MAX_SIG
  };

Now for each signal type we have to derive event structure from SST_Evt as base member with addional signal data. For this example we can select LED number assuming there is single AO controlling many LED on board.

We define event structure as

  typedef struct {
    SST_Evt super; /*< inherit from SST_evt */
    uint32_t pin;  /*< Pin number  */
  }LED_Evt;

Task Structure

Now we will create worker AO with initialize and AO task handler.

  typedef struct {
    SST_Task super;
  }LED_AO;

As we will be referring pointer to AO base class/struct, it is much better to create pointer reference for easy use.

  static LED_AO LED_AO_Instance;
  SST_Task * const ptrLED_AO = &LED_AO_Instance.super;
Define functions for AO 
  /* Counstroctor this is private function to be called from initialziation interface */
  static void LEDAO_ctor(LED_AO * const me);
  /* Initialie function with initial event */
  static void LEDAO_init(LED_AO * const me, SST_Evt const * const ie);
  /* Task function */
  static void LEDAO_dispatch(LED_AO * const me, SST_Evt const * const e);
Create SST tasks

We will bind the tasks init and dispatch functions

  static void LEDAO_ctor(LED_AO * const me)
  {
    SST_Task_ctor(&me->super,
		  (SST_Handler)&LEDAO_init,
		  (SST_Handler)&LEDAO_dispatch);
  }
Dispatcher

This function will be of prototype SST handler 

  static void LEDAO_dispatch(LED_AO * const me, SST_Evt const * const e)
  {
    switch(e->sig)
      {
      case LED_ON:
	break;
      case LED_OFF:
	break;
      }
  }

Interrupts Assignments

Based on microcontroller used we have to find unused interrupts for calling task handlers. In case of Nucleo-F303RE we can use PVD interrupt number for LED AO.

We should assign it using SST_Task_setIRQ function.

  SST_Task_setIRQ(LED_AO,PVD_IRQn);

Once set we have to bind this ISR call to our dispatch function.

  void PVD_IRQHandler(void)
  {
    SST_Task_activate(ptrLED_AO);
  }

Event storage

AO needs event queue for storing incoming event request. We have to create array of poin

  static SST_Evt const * LED_AOq[10];

AO Runtime creation

Once above things are setup its now time for putting AO in action. To do this we need to call SST_Task_start function as

  SST_Task_start(ptrLED_AO,1U,LED_AOq,ARRAY_NELEM(LED_AOq),(SST_Evt const *)0);

SST Callbacks (Hooks)

SST_onStart`

We need to provide hook function which get called from SST_Task_run function. For F303RE

  /* SST callbacks ===========================================================*/
  void SST_onStart(void) {
    SystemCoreClockUpdate();

    /* set up the SysTick timer to fire at BSP_TICKS_PER_SEC rate */
    SysTick_Config((SystemCoreClock / BSP_TICKS_PER_SEC) + 1U);

    /* set priorities of ISRs used in the system */
    NVIC_SetPriority(SysTick_IRQn, 0U);
    /* ... */
  }
  /*..........................................................................*/
  void SST_onIdle(void) {
  #ifdef NDEBUG
    __WFI(); /* Wait-For-Interrupt */
  #endif
  }

Supporting ISR

Apart from repurposed ISRs we also need basic system ISR for SST kernel for timebase and timer activities.

  void SysTick_Handler(void)
  { /* system clock tick ISR */
    SST_TimeEvt_tick(); /* process all SST time events */
  }

Start

Once all the setup are complete we can start scheduler by calling SST_Task_run This function shall never return.

  return SST_Task_run();

Main putting it all togethor 

  int main(void)
  {
    SST_init();
    System_Init();

    LEDAO_Initialize();

    static SST_Evt const *LED_AOq[10];

    SST_Task_start(ptrLED_AO, 1U, LED_AOq, ARRAY_NELEM(LED_AOq),
		   (SST_Evt const*) 0);

    return SST_Task_run();
  }

SST C++ Template application

C++ provides much natural feel of writing object oriented application as we don't have to be SUPER careful to make sure I am passing correct data.

Design

Design exercise remain same in C and C++. Only implmentation and inter mixing of C and C++ code should be taken care.

Active Object

All the actie objects should be derived from task class

  class LED_AO : public SST::Task {
  public:
  LED_AO(void);
  void init(SST::Evt const * const ie) override;
  void dispatch(SST::Evt const * const e) override;
  };

We don't require explicite constructor as C++ compiler takes care of it.

Events

For simple design events and signals are still same as C implmentation

  enum Signals {
    LED_ON,
    LED_OFF,
    MAX_SIG
  };
  struct {
    SST::SST_Evt super; /*< inherit from SST_evt */
    uint32_t pin;  /*< Pin number  */
  }LED_Evt;

Support for event queue

Basic SST c or C++ do not support event queue. As we are statically allocating event and pointers. Dispatch function should consume the event else there will be race condition and event data may get corrupt.

Hence I am thinkging of adding event pool so that we can buffer the messages . STM32 freeRTOS port gave away event pool against heap management. Which is nice feature RTOS like uCos provides.

Exploring options I think memory pool facility by uCos has very efficient algorithm but unfortunatly uCos supports Cortex M3 and above.

But here come rescue Keil MDK which Free Keil for STM32C0, STM32F0, STM32L0 and STM32G0 So for these cores we have to use Keil development. CubeMX supports generation of this code very easy. But I have to give away CMAKE at least for now.
